# Contributing

ATP is a protocol. Changes fall into three categories:

1. **Spec changes** — edits to `docs/SPEC.md`. Must be accompanied by a reference-implementation change and rationale in the PR description.
2. **Reference implementation** — `packages/*`. Must not diverge from the spec.
3. **Examples & docs** — low-friction; PRs welcome.

Before submitting a protocol change, open an issue with the `spec-proposal` label describing the problem, proposed change, and backwards-compatibility impact.
